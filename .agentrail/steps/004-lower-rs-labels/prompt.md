Phase 2 finish: label-aware lowering. Extend mlpl-lower-rs to handle annotation syntax (x : [batch, dim] = ...), the label(), labels(), relabel(), and reshape_labeled() builtins, and -- the key compile-time win -- resolve statically-known labels at macro-expansion time where possible. Specifically: when both operands of a matmul are ArrayLit with statically known LabeledShape, check the contraction axis at lower-time and emit a compile_error! instead of a runtime ArrayError::LabelMismatch. Fall back to runtime validation when either operand's shape/labels are not known statically. TDD: (a) a matmul with matching contraction labels lowers and runs; (b) a matmul with mismatched static labels produces a lower-time error whose message names both labels; (c) runtime-sourced shapes still go through mlpl-rt and error at runtime.
