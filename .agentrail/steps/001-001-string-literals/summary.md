Added string literals to parser (TokenKind::StrLit, Expr::StrLit, double-quoted with escapes \" \\ \n \t \r) and Value enum to mlpl-eval (Value::Array | Value::Str). New eval_program_value() returns Value for callers that need strings; eval_program() still returns DenseArray for backward compatibility, erroring if final value is a string. mlpl-wasm and mlpl-repl now use eval_program_value so strings round-trip through both REPLs. Split eval_ops module to satisfy function-count limit. 18 new tests (11 parser, 5 eval, 2 wasm) all passing.
