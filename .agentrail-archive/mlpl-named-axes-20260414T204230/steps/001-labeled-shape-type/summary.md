Phase 1 of Saga 11.5 (Named Axes) shipped. Added LabeledShape type to mlpl-core (not mlpl-array -- mlpl-core is already the shared-types crate that mlpl-array/mlpl-eval both depend on, and putting it there avoided a module-count sw-checklist regression). DenseArray gains a pub(crate) labels: Option<Vec<Option<String>>> field initialized to None at every existing constructor and op-result site, so default behavior is unchanged. The DenseArray::labels() accessor lives in indexing.rs alongside rank() and elem_count() rather than dense.rs, keeping dense.rs at its 7-function budget. The labels(x) builtin returns a Value::Str with a comma-joined encoding (positional axes -> empty string, labeled -> name) since MLPL has no string-vector literal yet; dispatch is a special case in eval_expr next to svg/grad/apply because mlpl_runtime::call_builtin can only return DenseArray. Deferred with_labels()/labeled_shape() helpers to Phase 2.
