Compile-to-rust step 006 shipped. New dev-only crate crates/mlpl-parity-tests/ runs 9 curated MLPL programs through both the interpreter (eval_program) and the compile path (lower + rustc + run), asserts numeric parity within f64 epsilon. Amortizes rustc cost by generating one big Rust program with one fn per case and compiling once. Gated by MLPL_PARITY_TESTS=1. Second test measures speedup on a 100x100 reshape/reduce workload: interpreter=479500ns, compiled=53000ns, 9.05x speedup (recorded for release-step summary). All 9 parity cases agree bit-for-bit. Model DSL / autograd / optimizer demos are out of compile scope (TensorCtor + Train + grad/adam not lowered) and deferred. All quality gates clean; sw-checklist unchanged. 617 tests total (+2 parity tests).
