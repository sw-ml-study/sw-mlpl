Phase 5: mlpl build foo.mlpl subcommand (native binary). Extend the existing mlpl-cli binary (or add a new mlpl-build bin if cleaner) with a build subcommand: mlpl build foo.mlpl -o foo reads MLPL source, generates a tiny temp cargo project (main.rs with fn main block inlining the mlpl! macro wrapping the user's source, Cargo.toml pointing at the workspace mlpl crate), shells out to cargo build --release --target-dir tmp, and moves the resulting binary to the requested path. Cross-compile support: forward a --target flag to cargo.
